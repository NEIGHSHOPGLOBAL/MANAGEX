# ManageX

### Simple Team Planner & Task Management System

---

# Project Vision

ManageX is a lightweight, modern Team Planner and Task Management platform built for organizations that want an easy-to-use system instead of a complex enterprise ERP.

The primary goal is quick task assignment, project management, verification workflow, and personal planning while maintaining a clean UI, fast performance, and excellent mobile experience.

The application will be built as a **Progressive Web App (PWA)** using **React.js** for the frontend and **Python Flask** for the backend.

# Task Categories

## 1. Personal Task

Visible only to the creator.

Purpose

Personal reminders and planning.

Features

* Title
* Description
* Due Date & Time
* Priority
* Status
* Checklist (optional)
* Personal Notes

No approval required.

No project association.

---

## 2. Normal Task

Can be assigned to anyone.

Purpose

Quick work assignments.

Flow

Assigned

↓

In Progress

↓

Done

No manager verification required.

Once employee marks Done

* Task becomes read-only.
* Only Super Admin can reopen or modify the completion state.

---

## 3. Project Task

Belongs to a project.

Flow

Assigned

↓

In Progress

↓

Done by Employee

↓

Pending Verification

↓

Verified by Management

↓

Completed

If rejected

↓

Returns back to employee.

Only authorized management can verify completion.

# Late Completion Logic

Every task has

* Due Date
* Due Time

If completed after deadline

Show

Late Completion

Important Rule

Late highlighting is applicable only for

* COO
* Branch Manager
* Employee

Exception

If Super Admin assigns any task to an Admin, late completion should also be tracked.

Tasks assigned directly to Super Admin are never considered late.

---

# Task Verification Rules

Personal Task

No verification.

Normal Task

Employee marks Done.

Task completes immediately.

Project Task

Employee marks Done.

Task enters Pending Verification.

Management verifies.

Task becomes Completed.

Only Super Admin can reopen completed tasks.

Admins cannot reopen completed tasks.
